QUESTION ANSWERED (问题回答):
  "这个分支的backend优雅停机是怎么实现的？"

ANSWER (答案):
  Backend通过ShutdownManager实现优雅停机，核心流程：
  1. SIGTERM信号 → 标记停机状态
  2. 拒绝新请求(503) + /api/ready返回503 → K8s停止发流量
  3. 等待活跃流完成(最多600秒)
  4. 超时强制取消 + 清理资源 → 优雅退出

================================================================================
  Key Features Documented (记录的关键特性)
================================================================================

5 Core Mechanisms (5个核心机制):
  1. State Management (状态管理)
     - ShutdownManager tracks global state
     - Cross-worker communication via Redis

  2. Stream Tracking (流追踪)
     - register_stream(subtask_id)
     - unregister_stream(subtask_id)
     - get_active_stream_count()

  3. Reject New Requests (拒绝新请求)
     - Middleware returns 503
     - /api/ready returns 503 → K8s stops traffic

  4. Graceful Wait (优雅等待)
     - Wait up to 600s for streams to complete
     - Streams detect is_shutting_down

  5. Force Cleanup (强制清理)
     - Timeout → cancel_all_streams()
     - Close resources → Exit

================================================================================
  Shutdown Flow (停机流程)
================================================================================

  SIGTERM Signal
      ↓
  [1] initiate_shutdown()
      → _shutting_down = True
      → Notify via Redis
      ↓
  [2] Middleware: Reject new requests (503)
      ↓
  [3] /api/ready: Return 503
      → K8s removes Pod from load balancer
      ↓
  [4] Wait for active streams (max 600s)
      → Streams check is_shutting_down
      → Exit gracefully when detected
      ↓
  [5] Timeout? → cancel_all_streams()
      ↓
  [6] Cleanup: HTTP client, jobs, telemetry
      ↓
  [7] Process exits (graceful shutdown complete)

================================================================================
  Configuration (配置)
================================================================================

Environment Variables:
  GRACEFUL_SHUTDOWN_TIMEOUT=600        # Default: 10 minutes
  SHUTDOWN_REJECT_NEW_REQUESTS=true    # Default: true

Kubernetes Configuration:
  readinessProbe:
    httpGet:
      path: /api/ready                 # Returns 503 during shutdown
      port: 8000
  terminationGracePeriodSeconds: 660   # TIMEOUT + 60s buffer

================================================================================

  ----------------------------------------------------------------
  /api/health     Liveness     200       200 (still alive)
  /api/ready      Readiness    200       503 (stop traffic)
  /api/startup    Startup      200       200 (startup done)
