# Wegent 与 Wework 文档重组计划

## 目标

将 Wegent 和 Wework 建设为同一文档站中的两个并列产品文档空间。用户可以从网站顶部直接切换产品，并在切换语言后继续浏览相同产品的对应语言文档。

## 信息架构

源文档按语言和产品组织：

```text
docs/
├── en/
│   ├── wegent/
│   └── wework/
└── zh/
    ├── wegent/
    └── wework/
```

发布路径：

```text
/docs/wegent/...
/docs/wework/...
/docs/en/wegent/...
/docs/en/wework/...
```

Wegent 文档覆盖平台安装部署、智能体、对话、Feed、待办、知识库、集成、平台设置、API 和服务端开发。

Wework 文档覆盖桌面工作台安装、项目、任务、AI 协作、代码工作台、内置浏览器、设备、插件、设置、故障排查和桌面端开发。

## 内容原则

1. 以用户要完成的工作组织操作文档，界面名称与当前产品保持一致。
2. 用户指南只保留会影响操作、选择、结果和数据安全的信息。
3. 实现原理、协议、状态机、测试和发布流程归入对应产品的开发者指南。
4. 一个主题只保留一个主要说明页面；跨产品能力通过链接关联，不重复维护。
5. 中文和英文目录保持一致，新增内容先完成中文，再补齐英文。
6. 所有页面提供 Docusaurus frontmatter，并由目录分类和 `sidebar_position` 控制导航顺序。

## 实施步骤

### 1. 重组源文档

- 将现有 Wegent 文档移入 `docs/{en,zh}/wegent/`。
- 建立 `docs/{en,zh}/wework/`。
- 修复迁移产生的相对链接、首页入口和分类配置。

### 2. 建设 Wework 文档

- 快速开始：产品简介、安装、首次启动和第一个任务。
- 项目：本地目录、Git 项目、工作区和 Worktree。
- 任务：模型与设备选择、附件、Skills、任务控制和会话管理。
- 代码工作台：文件、终端、变更审核、撤销和 Git 工作流。
- 内置浏览器：网页访问、截图、批注和数据设置。
- 设备与云端工作：本机设备、Wegent 云端、云设备和远程设备。
- 插件与 Skills：安装、管理和任务内使用。
- 设置与故障排查：模型、代理、快捷键、外观、数据和常见问题。
- 开发者指南：桌面架构、本地运行时、浏览器、验证、发布和性能诊断。

### 3. 整理 Wegent 文档

- 更新首页和用户指南索引，移除过期目录引用。
- 将 Wework 专属内容迁出 Wegent 的用户指南和开发者指南。
- 保留与 Wework 对接所需的平台管理说明，并链接到对应 Wework 页面。

### 4. 更新发布站

- 为 Wegent 和 Wework 配置独立的自动生成侧边栏。
- 在顶部导航加入产品切换入口。
- 更新首页，使用户先选择产品再进入文档。
- 调整同步工作流，完整同步目录并清理已经删除的页面和分类文件。

### 5. 验证

- 构建中文和英文 Docusaurus 站点。
- 检查断链、重复文档 ID、无效 MDX、图片和示例路径。
- 在 Wework 内置浏览器中打开本地站点，验证首页、产品切换、侧边栏、语言切换和代表性页面。

## 完成标准

- 文档网站能够在 Wegent 和 Wework 文档之间直接切换。
- 两个产品拥有独立、可扩展的目录和侧边栏。
- 中文和英文均可成功构建。
- Wework 用户可以从安装开始完成项目创建、任务执行、变更审核和故障处理。
- Wegent 文档不再将 Wework 表述为附属功能。
- 发布仓库同步后不会保留源仓库中已经删除的旧页面或分类。
