## üìì Notebook Mode

Notebook mode is designed for interactive research and Q&A sessions.

### Features

- **Built-in Chat**: Chat directly within the knowledge base
- **Document Selection**: Choose specific documents for context
- **Three-column Layout**: Documents, content, and chat side by side
- **Auto-selection**: New documents automatically selected

### Use Cases

| Scenario | Description |
|----------|-------------|
| Research | Interactive exploration of documents |
| Q&A Sessions | Quick answers from specific documents |
| Learning | Study materials with AI assistance |
| Small Projects | Focused document collections |

### Document Selection

In Notebook mode, you can select which documents to include in the chat context:

1. View the document list
2. Check/uncheck documents to include
3. Use **Select All** / **Deselect All** for batch operations
4. Selected documents provide context for AI responses

---

## üìö Classic Mode

Classic mode is designed for large document collections and archives.

### Features

- **Unlimited Documents**: No document count limit
- **Document List View**: Focus on document management
- **Batch Operations**: Efficient bulk document handling
- **Archive Friendly**: Suitable for large collections

### Use Cases

| Scenario | Description |
|----------|-------------|
| Documentation | Large technical documentation |
| Archives | Historical document storage |
| Enterprise | Company-wide knowledge repositories |
| Reference | Comprehensive reference materials |

---

## üîÑ Choosing the Right Type

### Choose Notebook Mode When:

- You have fewer than 50 documents
- You need interactive chat within the knowledge base
- You want to select specific documents for context
- You're doing research or Q&A sessions

### Choose Classic Mode When:

- You have more than 50 documents
- You primarily need document storage and retrieval
- You're building a large archive
- You'll access via agents rather than direct chat

---

## ‚ö†Ô∏è Important Notes

### Type Cannot Be Changed

Once a knowledge base is created, its type cannot be changed. Choose carefully based on your needs.

### Document Limit Enforcement

Notebook mode enforces a 50-document limit. If you need more documents, create a Classic mode knowledge base.

### Migration

To migrate from Notebook to Classic (or vice versa):

1. Create a new knowledge base with the desired type
2. Re-upload documents to the new knowledge base
3. Delete the old knowledge base (optional)
