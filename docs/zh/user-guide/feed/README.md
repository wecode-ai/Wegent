# 概述

Feed（关注）是 Wegent 的自动化任务执行功能，让 AI 智能体按照设定的时间或事件自动执行任务，并以类似社交媒体动态的形式展示执行结果。

---

## 📋 核心概念

### 什么是订阅器

**订阅器（Subscription）** 是 Feed 功能的核心，它定义了：

- **执行什么**：要执行的 Prompt 模板
- **谁来执行**：选择哪个智能体
- **何时执行**：触发条件（定时、间隔、事件等）
- **如何执行**：超时、重试等配置

```
订阅器 = Prompt 模板 + 智能体 + 触发条件 + 执行配置
```

### 执行记录

每次订阅器触发后会产生一条**执行记录（Execution）**，包含：

- 执行状态（等待中、运行中、已完成、失败等）
- AI 生成的结果摘要
- 完整的对话历史
- 执行时间和耗时

---

## 🎯 主要功能

### 1. 动态时间线

以社交媒体风格展示所有订阅器的执行动态：

- 按时间分组（今天、昨天、本周、更早）
- 实时更新执行状态
- 展开查看 AI 生成的摘要
- 快速跳转到完整对话

### 2. 订阅器管理

创建和管理自己的订阅器：

- 多种触发方式（定时、间隔、一次性、Webhook）
- 灵活的 Prompt 模板（支持变量）
- 可选的代码仓库配置
- 模型覆盖选项

### 3. 发现与关注

发现其他用户公开的订阅器：

- 浏览公开订阅器
- 关注感兴趣的订阅器
- 在动态中查看关注订阅器的执行结果

### 4. 订阅器市场

租用市场中的订阅器：

- 浏览市场订阅器
- 自定义触发配置
- 使用自己的模型执行

## 🚀 快速开始

### 创建第一个订阅器

1. 进入 **Feed** 页面
2. 点击 **新建订阅器**
3. 填写基本信息：
   - 名称：如"每日新闻摘要"
   - 选择智能体
   - 设置触发方式（如每天上午 9 点）
   - 编写 Prompt 模板
4. 点击创建

### 查看执行结果

创建后，订阅器会按设定时间自动执行：

1. 在 **全部** 标签页查看动态时间线
2. 点击执行记录查看 AI 摘要
3. 点击"查看对话"查看完整内容

## 💡 使用场景

### 信息采集类

- **每日新闻摘要**：每天早上汇总行业新闻
- **竞品监控**：定期分析竞品动态
- **数据报告**：每周生成数据分析报告

### 执行类

- **代码审查**：定时检查代码质量
- **文档更新**：自动更新项目文档
- **自动化测试**：定期运行测试并报告结果

### 事件驱动

- **Webhook 集成**：接收外部系统通知并处理
- **Git Push 触发**：代码提交后自动执行任务
