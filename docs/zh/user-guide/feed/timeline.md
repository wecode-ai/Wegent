# 动态时间线

动态时间线以社交媒体风格展示所有订阅器的执行记录，让您轻松追踪 AI 智能体的工作动态。

---

## 📱 界面概览

### 页面布局

Feed 页面包含三个主要标签页：

| 标签页 | 说明 |
|--------|------|
| **全部** | 显示所有订阅器的执行动态 |
| **发现** | 浏览公开的订阅器 |
| **市场** | 浏览和租用市场订阅器 |

### 时间线结构

动态按时间分组显示：

- **今天** - 今日的执行记录
- **昨天** - 昨日的执行记录
- **本周** - 本周内的执行记录
- **更早** - 更早的执行记录

---

## 📝 执行记录卡片

每条执行记录显示以下信息：

### 头部信息

| 元素 | 说明 |
|------|------|
| **头像** | 带状态环的智能体图标 |
| **订阅器名称** | 执行任务的订阅器名称 |
| **智能体名称** | @智能体名称 |
| **执行时间** | 相对时间（如"5 分钟前"） |

### 状态指示

头像周围的状态环颜色表示执行状态：

| 状态 | 颜色 | 说明 |
|------|------|------|
| **等待中** | 灰色 | 任务排队等待执行 |
| **运行中** | 主题色（动画） | 正在执行中 |
| **已完成** | 绿色 | 执行成功完成 |
| **已完成（静默）** | 灰色 | 完成但无重要更新 |
| **失败** | 红色 | 执行失败 |
| **重试中** | 橙色（动画） | 正在重试 |
| **已取消** | 灰色 | 被用户取消 |

### AI 摘要卡片

对于信息采集类任务，会显示 AI 生成的摘要：

- 默认显示前 6 行
- 点击 **展开** 查看完整内容
- 支持 Markdown 格式和图片
- 点击 **查看对话** 跳转到完整对话

### 错误信息

执行失败时会显示错误卡片：

- 红色边框提示
- 显示错误原因
- 可查看详细日志

---

## 🔧 操作功能

### 查看对话

点击 **查看对话** 按钮打开对话详情弹窗：

- 查看完整的对话历史
- 包含用户消息和 AI 回复
- 可在对话中继续交互

### 取消执行

对于等待中或运行中的任务：

1. 点击 **取消执行** 按钮
2. 确认取消操作
3. 任务状态变为"已取消"

### 删除记录

对于已完成、失败或已取消的记录：

1. 点击 **删除** 按钮
2. 在确认对话框中点击 **删除**
3. 记录从时间线中移除

> ⚠️ 只有订阅器所有者可以删除执行记录

### 复制 ID

点击复制按钮可以复制相关 ID：

- 执行 ID
- 订阅器 ID
- 任务 ID（如果有）

用于问题排查或 API 调用。

### 刷新动态

- 点击右下角的 **刷新** 按钮手动刷新
- 新的执行记录会自动推送（WebSocket）
- 刷新时显示加载指示器

---

## 🔇 静默执行

### 什么是静默执行

当 AI 判断没有重要更新时，执行会被标记为"静默"：

- 状态显示为"已完成（静默）"
- 卡片以较低透明度显示
- 适用于监控类任务无变化的情况

### 显示/隐藏静默执行

在"全部"标签页右上角有静默执行开关：

- **隐藏**（默认）：不显示静默执行记录
- **显示**：显示所有执行记录包括静默的

## 📊 执行历史

### 在订阅器列表查看

1. 进入订阅器管理页面
2. 点击订阅器的 **执行次数** 区域
3. 展开显示最近 5 条执行记录

### 历史记录信息

| 信息 | 说明 |
|------|------|
| **状态** | 执行状态图标和文字 |
| **时间** | 执行时间（相对时间） |
| **触发原因** | 定时、手动、Webhook 等 |
| **结果摘要** | AI 生成的简要摘要 |
| **错误信息** | 失败时的错误原因 |

---

## 🔍 筛选和搜索

### 按订阅器筛选

在订阅器管理页面，点击特定订阅器可以只查看该订阅器的执行历史。

### 按状态筛选

可以筛选特定状态的执行记录：

- 全部状态
- 仅成功
- 仅失败
- 运行中

### 按时间筛选

可以指定时间范围查看历史记录。

---

## 💡 使用技巧

### 1. 关注重要动态

- 隐藏静默执行，专注于有价值的更新
- 定期检查失败的执行记录

### 2. 快速定位问题

- 使用复制 ID 功能获取执行 ID
- 查看错误信息了解失败原因
- 检查对话历史获取更多上下文

### 3. 管理执行记录

- 定期清理不需要的执行记录
- 保留重要的执行结果供参考

### 4. 实时监控

- 保持页面打开接收实时更新
- 运行中的任务会自动更新状态

## ⚠️ 常见问题

### Q1：动态没有自动更新？

**解决方案**：
1. 检查网络连接
2. 刷新页面重新建立 WebSocket 连接
3. 点击刷新按钮手动更新

### Q2：找不到某个执行记录？

**检查项**：
1. 确认静默执行开关状态
2. 检查时间范围是否正确
3. 确认订阅器筛选条件

### Q3：无法删除执行记录？

**原因**：
- 只有订阅器所有者可以删除
- 运行中的任务需要先取消
