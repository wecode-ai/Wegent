# 概述

资源库用于发布、发现和接受可复用资源。发现页面向“可复用方案”，团队可以把已经配置好的智能体或技能包装成面向场景的方案说明，其他用户查看用法后即可接受到自己的资源列表。

---

## 核心概念

| 概念 | 说明 |
|------|------|
| 发现 | 面向所有用户的方案市场页，用于按目标和场景浏览团队发布的资源说明 |
| 我的 | 当前用户可使用和可管理的资源列表 |
| 发布 | 将自己的智能体或技能发布为资源说明 |
| 接受 | 接受发布者的分享，让该资源进入自己的资源列表 |
| 发现助手 | 通过对话帮助用户搜索和筛选资源的智能体 |

资源库展示的是资源说明，不会复制发布者的智能体或技能。发布者继续编辑原始资源时，接受该资源的用户会看到发布者共享内容的最新效果。

---

## 使用发现页

1. 从侧边栏进入 **更多** → **资源库**。
2. 默认进入 **发现** 标签页。
3. 在搜索框中输入目标、场景或关键词，查找匹配方案。
4. 通过卡片上的类型标识确认它背后是智能体还是技能。
5. 点击资源卡片右下角的查看用法按钮，打开 mini 使用指南。
6. 确认方案说明和开始步骤后，点击接受按钮完成分享接受。

接受资源后，按钮会显示已接受。该资源会进入 **我的** 页中对应类型的列表。

---

## 使用发现助手

发现助手是一个实际的 Wegent 智能体，而不是前端静态搜索。它会通过已有知识库检索工具查询组织级资源库知识库，然后根据当前可发现的资源说明给出建议。

适合向发现助手提出的问题：

- “有没有适合写周报的智能体？”
- “我想做代码评审，应该用哪个资源？”
- “帮我找一个能处理知识库文档的技能。”

如果发现助手提示没有找到资源，通常说明组织级资源库知识库未配置、资源还未同步完成，或当前关键词与资源说明不匹配。

---

## 发布资源

在 **我的** 页中，资源拥有者可以从智能体或技能列表发布资源：

1. 找到要发布的智能体或技能。
2. 点击发布按钮。
3. 填写展示名称、描述、标签和版本号。
4. 提交后，资源会出现在发现页。

发布、更新或下架资源时，系统会同步一份展示说明到组织级资源库知识库，供发现页和发现助手检索。

---

## 管理我的资源

**我的** 页保留管理式布局，适合批量查看和维护资源。可以按资源类型、来源和模式筛选：

- 智能体、技能、模型、执行器、检索器等资源类型
- 我创建的、团队、系统和资源库来源
- 聊天、编码、设备等智能体模式

资源库来源的技能或智能体不会自动默认启用。需要默认启用时，请在对应资源管理页手动设置。

---

## 管理员配置

发现页由 `ResourceLibraryDiscoveryConfig` Kind 驱动。管理员需要准备一个组织级知识库，通常命名为 `资源库`，并配置发现助手 Team。

详细 YAML 字段见 [YAML 定义格式](../../reference/yaml-specification.md#-resourcelibrarydiscoveryconfig)。
