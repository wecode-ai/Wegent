# 概述

Integrations（集成）是 Wegent 与外部系统的连接功能，让您可以将 AI 能力集成到现有的工作流程和工具中。

---

## 📋 核心概念

### 集成类型

Wegent 支持多种集成方式：

| 类型 | 说明 | 用途 |
|------|------|------|
| **IM 渠道** | 与即时通讯工具集成 | Slack、Discord、企业微信等 |
| **Webhook** | 接收外部系统的事件通知 | CI/CD、监控系统 |
| **API** | 程序化调用 Wegent 功能 | 自定义应用集成 |

### 集成工作流

```
外部系统 → 触发事件 → Wegent 处理 → AI 执行 → 返回结果
```

---

## 🎯 主要功能

### 1. IM 渠道集成

将 AI 智能体接入即时通讯工具：

- 在聊天频道中与 AI 对话
- 接收 AI 的主动通知
- 支持群组和私聊模式
- 命令式交互

### 2. Webhook 集成

接收外部系统的事件：

- Git 仓库事件（Push、PR、Issue）
- CI/CD 流水线状态
- 监控告警
- 自定义事件

### 3. 双向通信

支持与外部系统的双向交互：

- 接收外部触发
- 调用外部 API
- 发送通知消息
- 状态同步

## 🚀 快速开始

### 配置 IM 渠道集成

1. 进入 **Integrations** 页面
2. 选择 **IM 渠道**
3. 选择要集成的平台（如 Slack）
4. 按照指引完成授权配置
5. 选择要接入的智能体
6. 开始使用

### 配置 Webhook

1. 在 **Feed** 中创建订阅器
2. 选择 **事件触发** 类型
3. 获取 Webhook URL
4. 在外部系统中配置 Webhook
5. 测试触发

## 💡 使用场景

### 开发工作流

- **代码审查**：Git Push 自动触发代码审查
- **部署通知**：CI/CD 完成通知
- **Issue 处理**：自动分析和回复 Issue

### 运维监控

- **告警处理**：接收监控告警并自动分析
- **状态报告**：定时发送系统状态报告
- **故障排查**：协助诊断系统问题

### 团队协作

- **日常问答**：在 IM 中询问 AI
- **知识查询**：查询文档和知识库
- **任务提醒**：定时任务和提醒
