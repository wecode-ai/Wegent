# 桌面应用偏好

Wework 桌面版的“设置 → 通用”用于控制应用窗口和后台运行行为。这些偏好只影响本机桌面应用，不会改变云端任务、模型或执行器配置。

## 界面语言

“界面语言”用于切换 Wework 桌面界面和系统托盘菜单使用的语言。可以选择“系统”“中文”或“EN”。

选择“系统”时，Wework 会根据当前系统语言自动使用中文或英文。手动选择“中文”或“EN”后，Wework 会保存该偏好，并在下次启动时继续使用。

## 启动后是否显示主界面

开启“启动应用后显示主界面”后，启动 Wework 会自动打开主窗口。关闭后，应用启动时只进入系统托盘，适合希望开机后让任务能力常驻后台的用户。

当主界面未显示时，可以单击系统托盘图标重新打开窗口。

## 从其他 AI 应用导入

“设置 → 通用 → 从其他 AI 应用导入工作内容”可以把本机其他编码应用中的兼容内容导入 Wework 独立的 Codex 目录。导入会覆盖目标目录中的同名文件，但不会删除来源应用中的内容。

- **Codex**：导入范围与 Wework 首次启动时的 Codex 迁移一致，包括配置、登录信息、全局指令、插件、Skills 和相关缓存。
- **Claude Code**：将 `~/.claude/CLAUDE.md` 作为 Codex 全局 `AGENTS.md` 导入，并导入 `~/.claude/skills/`。

第一期不导入项目列表、Claude Code 插件或历史会话。导入功能只在 Wework 桌面版中可用；如果来源目录不存在或没有可兼容内容，Wework 会保留弹窗并显示错误，可以修复来源内容后重试。

## 关闭窗口后后台运行

“关闭主窗口后继续在后台运行”默认开启。开启后，点击窗口关闭按钮不会退出 Wework；正在执行的任务会继续运行，应用会保留在系统托盘中。

第一次关闭主窗口时，Wework 会提示“关闭窗口后，后台可以继续执行任务”。用户可以选择保持打开，或确认转入后台。确认后，该说明不会在每次关闭时重复出现。

关闭到后台时，Wework 会销毁主窗口的 WebView，以释放主界面相关资源；任务执行器不会因此停止。重新从托盘打开窗口时，Wework 会创建新的主窗口并恢复到当前任务状态。

## 退出应用

如果需要完全退出 Wework，可以从系统托盘菜单选择退出。退出应用会停止本机运行中的执行器进程。

如果关闭“关闭主窗口后继续在后台运行”，点击窗口关闭按钮会退出应用，而不是进入托盘后台。
