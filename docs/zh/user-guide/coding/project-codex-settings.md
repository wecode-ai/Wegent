# 项目 Codex 设置

Wework 桌面端可以为每个本地项目单独配置 Codex 指令和插件。打开项目菜单并选择“项目设置”，即可通过图形界面编辑 Codex 原生项目文件。

## 与 Codex 原生配置的关系

项目设置不会创建 Wework 专用的配置格式：

- “项目指令”直接读写项目根目录的 `AGENTS.md`。Codex 会按照自身的指令发现和层级规则，将它与用户级及上级目录中的指令共同使用。
- “Codex 配置”直接读写项目根目录的 `.codex/config.toml`。
- 项目插件会写入 `.codex/config.toml` 的 `[plugins."<plugin-key>"]` 配置。它们追加到全局已启用插件，而不是覆盖全局插件。
- 插件安装包由本机 Codex 共享；选择“安装到此项目”时，Wework 安装共享插件、保持其全局关闭，并在当前项目中启用。

因此，在 Wework 中完成的设置也可以被项目目录中直接运行的 Codex 识别；手工修改这些原生文件后，Wework 重新打开页面也会显示最新内容。

## 生效时机

保存会以文件修订号检查并发修改，避免静默覆盖磁盘上的新内容。Codex 会在新会话启动时加载项目指令、配置和插件；已经运行的会话不会被项目设置页面强制重启。

提交 `AGENTS.md` 或 `.codex/config.toml` 前，请确认其中不包含密钥、令牌或仅适用于个人环境的路径。如果这些文件应由团队共享，可以像普通项目文件一样纳入 Git。
