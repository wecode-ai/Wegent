# 故障排查

## 模型不可用

- 在 **设置 → 模型** 检查 Codex 登录状态或本地模型地址。
- 确认选择的模型可以在当前设备运行。
- 使用代理时，保存设置后按提示重启 Codex。

## 项目无法打开

- 确认目录仍然存在且当前用户有读写权限。
- 远程项目应确认设备在线，并重新选择该设备上的目录。
- Git 项目目标目录已存在时，使用现有目录方式添加，或选择新的目标目录。

## Git 克隆失败

| 提示 | 检查内容 |
| --- | --- |
| Authentication failed | Git Token 是否有效、是否有仓库权限 |
| Repository not found | 仓库地址和当前账号权限 |
| Remote branch not found | 默认分支是否存在 |
| Could not resolve host / timeout | 执行设备的网络和代理 |

## 任务或终端无法运行

- 查看当前设备状态；离线设备恢复连接后再试。
- 设备提示升级时，先完成升级。
- 切换任务后，确认终端对应的是当前项目或 Worktree。
- 重新打开应用后仍失败，可完全退出 Wework 再启动。

## 无法审核或撤销变更

审核和撤销需要原执行设备在线，并保留该轮任务的变更记录。后续修改与原变更冲突时，先手动处理当前工作区，再决定是否回退。

## 内置浏览器异常

- 确认地址包含 `http://` 或 `https://`。
- 本地服务应确认进程和端口正在监听。
- 登录状态异常时，可以在 **设置 → 浏览器** 清理数据后重新登录。

## 云端连接失效

点击侧边栏的云端状态重新授权。确认 Backend 地址来自团队管理员，并且浏览器能够访问登录页面。

## 获取帮助

反馈问题时，请提供 Wework 版本、操作系统、复现步骤、界面错误信息和相关日志。不要在截图或日志中包含 Token、API Key、远程设备启动命令或认证文件内容。
