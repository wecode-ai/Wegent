# 项目与工作区

项目用于把同一份代码目录下的多次任务组织在一起。每个项目可以关联一个或多个执行设备上的目录。

## 使用现有目录

本机项目选择 **使用现有目录** 后，Wework 会打开系统目录选择器。再次添加同一目录时，Wework 会复用已有项目；如果项目曾被移除，则会恢复项目及其历史会话。

从项目列表移除本机项目不会删除磁盘目录。

## 从输入区创建项目

在新对话输入框上方打开项目选择器，可以新建空白项目或添加现有目录。创建完成后，新项目会同时出现在左侧项目列表和输入框上方，并立即成为当前新对话的项目工作区。

如果先切换到不使用项目的普通对话，再点击左侧对应项目的 **新对话**，Wework 会创建一个新的项目对话并重新选中该项目，不会复用普通对话的输入或会话状态。

## 创建 Git 项目

1. 新建项目并选择 **Git 仓库**。
2. 选择执行设备。
3. 选择仓库和默认分支。
4. 确认项目名称和目标目录。
5. 等待克隆完成。

如果仓库列表为空，在设置中检查 Git 服务连接和 Token 权限。目标目录已存在时，请选择现有目录，或更换新项目的目标目录。

## 普通工作区与 Worktree

- **普通工作区**：任务直接使用项目目录，适合串行处理和小规模修改。
- **Worktree**：每个任务使用独立的 Git 工作树，适合并行开发和需要隔离分支的任务。

使用 Worktree 时，文件树、终端和内置开发工具都会打开当前任务的工作树路径。归档任务前应先提交或保存仍需保留的修改。

## 在不同设备上使用项目

同一个项目可以关联本机、云设备或远程设备上的不同目录。新建任务时选择实际存放代码的设备。复制任务到另一台设备时，需要为目标设备选择对应的项目目录。

## 从终端打开项目

macOS 桌面版会安装 `wework` 命令。可以在终端中运行：

```bash
wework .
wework /path/to/project
```

Wework 会打开指定目录；应用已经运行时会复用现有窗口。
