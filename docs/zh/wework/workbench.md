# 代码工作台

## 文件与终端

右侧工作区用于浏览项目文件、预览内容和审核变更。助手回复中的本地文件链接可以直接打开对应文件；带行号的链接会定位到指定代码。

使用 `Command+J` 打开或关闭底部工作区面板。打开面板不会自动创建新终端；已有终端会按任务保留，切回任务时直接恢复。

点击底部标签栏的 **+**，可以根据当前设备能力选择 **Terminal**、**IDE** 或 **桌面**。Terminal 从当前项目目录启动，Worktree 任务从对应工作树启动；IDE 在系统默认浏览器中打开；桌面能力可用时在 Wework 内置浏览器中展示。

## 查看执行过程

任务运行时，对话会显示正在使用的工具和当前进度。已完成的工具调用会折叠为摘要，可以展开查看命令或修改过的文件。

## 浏览长对话

当对话内容超过当前视口时，消息区左侧会显示历史发言导航。导航固定在对话视口中央，不会随消息内容滚动；点击标记可跳转到对应发言，将指针悬停在标记上可预览用户请求和助手回复摘要。

## 切换对话与恢复位置

桌面工作台会缓存最近使用的对话页面。切换对话时，运行状态、最近消息、右侧工作区标签和面板状态会保持不变；切回后会立即显示离开时的工作区。普通对话页面最多缓存 10 个，超过上限后按最近使用顺序淘汰。

包含运行中 Terminal 或 Wework 内置浏览器的对话不会被普通缓存上限淘汰。Terminal 进程、标签和输出缓冲会继续保留，内置浏览器的页面、地址和标签状态也会保持；关闭对应资源后，该对话重新遵循普通缓存规则。隐藏对话不会响应当前对话的快捷键或浏览器打开事件。

首次打开的对话会定位到最新消息。已经停留在底部的对话切回后仍保持底部；如果之前浏览到历史消息，则恢复与底部的相对距离。长对话只挂载视口附近的消息，并复用已测量的消息高度，减少滚动时的 WebView 内存增长。

消息、滚动位置、虚拟测量和对话页面缓存都有数量上限。归档任务时会立即移除该任务的缓存；再次打开已被缓存淘汰的任务时，Wework 会从本地运行时重新读取完整记录。

## 使用回复中的选中文本

在助手回复中选择一段文本后，可以将其添加到当前对话输入框，或在侧边栏中基于该文本继续提问。回复仍在流式输出时也可以使用这些操作；后续内容更新不会关闭已经打开的文本操作菜单。

## 审核本轮变更

支持的 Git 任务完成后，助手消息下方会显示本轮文件变更，包括文件数量和增删行数。点击 **审核** 可查看完整 Diff、筛选文件、切换自动换行和复制 `git apply` 命令。

审核内容从执行任务的设备读取，因此设备需要保持在线。

## 撤销本轮变更

点击 **撤销** 后，Wework 会先检查反向补丁能否安全应用。后续修改与本轮内容冲突时，工作区不会被强制覆盖。撤销只处理当前轮产生的变更。

## 完成 Git 工作流

在提交或创建 Pull Request 前建议：

1. 审核所有变更文件。
2. 运行与改动相关的测试和格式检查。
3. 检查工作区是否包含临时文件或敏感信息。
4. 确认提交信息和目标分支。
5. 推送分支并创建 Pull Request。

归档 Worktree 任务前，先处理未提交修改；强制归档会移除对应工作树。
