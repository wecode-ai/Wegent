# Wework 启动动画

## 目标

消除 Wework 启动时 React 页面挂载和认证状态恢复期间的空白窗口，改为显示品牌 Logo 描边动画。

## 视觉与时序

启动层显示透明背景的蓝色 `We >_` Logo，动画顺序为：

1. `W` 描边出现
2. `e` 描边出现
3. `>` 描边出现
4. `_` 描边出现
5. 描边过渡为蓝色实心 Logo
6. 页面准备好后整体淡出

完整动画目标时长为 1.4 至 1.8 秒。页面较早准备好时，至少播放到 Logo 完整出现，避免启动层闪烁；页面较晚准备好时，Logo 保持完整状态直至应用就绪。

## 实现方式

- 启动层直接内嵌在 `wework/index.html`，确保 JavaScript 和 React 尚未加载时即可显示。
- Logo 使用从当前品牌图形提取的 SVG 路径。
- 描边动画使用 CSS `stroke-dasharray` 和 `stroke-dashoffset`。
- React 挂载后继续保留启动层，直到认证状态恢复并确定首个页面。
- 应用就绪时添加退出状态，完成淡出后移除启动层节点。
- 浅色和深色模式均使用各自页面背景色，避免闪白。
- `prefers-reduced-motion: reduce` 下跳过描边过程，直接显示完整 Logo，并在页面就绪时淡出。

## 就绪信号

就绪信号由应用路由层发出：

- 登录页或 OIDC 回调页完成首次渲染。
- 认证加载结束，并已确定登录用户或未登录状态。
- 已登录用户的应用外壳完成首次渲染。

启动层不等待工作台后续网络请求，避免接口异常导致启动动画永久停留。

## 验证

- 冷启动时不出现纯白空白窗口。
- 动画顺序为 `W → e → > → _ → 填充`。
- 登录、已登录和 OIDC 回调路径均能移除启动层。
- 认证请求缓慢时启动层保持，不发生内容闪烁。
- 减少动画模式下无描边动画。
- macOS、Windows 和 iOS WebView 均可正常显示。
