# 智能体简洁编辑设计

## 目标

优化智能体新建和编辑体验。默认展示简洁模式，让用户用更少概念完成一个 solo 智能体配置，同时保留现有高级能力。

简洁模式不是功能删减，而是重新组织高频配置。模型、技能、知识库、Prompt 都属于默认简洁配置；协作模式、多机器人、MCP、Dify 细节、模型白名单等放入高级配置。

## 当前状态

`TeamEditDialog` 当前由三块组成：

- `TeamBasicInfoForm`：名称、显示名称、描述、绑定模式、是否需要仓库。
- `TeamModeSelector`：solo、pipeline、route、coordinate、collaborate。
- `TeamModeEditor`：根据协作模式展示 `SoloModeEditor`、`PipelineModeEditor` 或 `LeaderModeEditor`。

solo 保存路径已经存在：先保存一个 Bot，再保存 Team，并设置 `workflow.mode = "solo"` 和 `leader_bot_id`。当前主要问题是编辑界面暴露了太多底层概念，绑定模式选中态不清楚，执行器列表直接显示 Shell，不符合用户理解。

## 设计原则

- 默认简洁：新建和 solo 智能体编辑默认进入简洁界面。
- 数据结构不变：继续保存为现有 `Team + Bot`，默认 `workflow.mode = "solo"`。
- 表单可重写：不强制复用现有 `BotEdit` 界面；可以重构或替换它，但必须保留保存契约。
- 高级可达：现有复杂能力放入高级区域，不因为默认简洁而丢失。
- 解释前置：绑定模式和执行器选择要有短说明，用户能判断何时选择。

## 简洁界面

默认区域包含：

- 基础信息：图标、智能体名称、显示名称、描述。
- 绑定模式：只保留 `对话`、`编码`、`设备`，默认选中 `对话`。
- 执行器：只展示 `简单`、`复杂`、`自定义` 三类。
- 模型：模型选择保留在简洁区。
- 技能：技能选择和管理入口保留在简洁区。
- 知识库：默认知识库选择保留在简洁区。
- Prompt：系统提示词保留在简洁区，并保留 Prompt 优化入口。

绑定模式改为复选卡片，每个选项包含标题、说明和明确的选中状态：

- `对话`：用于普通聊天和问答，默认选中。
- `编码`：用于代码仓库相关任务。选中后保留“需要仓库”开关。
- `设备`：用于本地设备或命令类任务。

执行器改为三类入口：

- `简单`：对应 Chat Shell，适合聊天、知识问答、轻量任务。
- `复杂`：对应 ClaudeCode Shell，适合编码、仓库操作和设备任务。
- `自定义`：展示用户或群组自定义 Shell 下拉。

当绑定模式包含 `编码` 或 `设备` 时，执行器只能选择支持 ClaudeCode 的选项。界面应直接解释原因，而不是在保存时才报错。

## 高级界面

高级区域默认折叠，包含现有低频和复杂配置：

- 协作模式：solo、pipeline、route、coordinate、collaborate，默认 solo。
- 多机器人成员配置和领导者选择。
- MCP 配置。
- Dify 专用配置。
- 自定义模型 JSON 配置。
- 模型白名单。
- 机器人复用、切换、克隆等现有能力。

高级区域可以继续复用现有组件，也可以在实现中逐步替换。关键约束是不要让高级能力阻塞默认简洁路径。

## 组件结构

新增默认简洁编辑组件，建议命名为 `SimpleTeamEditForm`。`TeamEditDialog` 在新建和编辑 solo 智能体时默认渲染它；编辑已有非 solo 智能体时默认展开高级配置，避免静默降级。

建议拆分子组件：

- `TeamIdentitySection`：图标、名称、显示名称、描述。
- `TeamBindModeCards`：三个绑定模式复选卡片。
- `ExecutorModeSelector`：简单、复杂、自定义执行器选择。
- `BotCoreConfigForm`：模型、技能、知识库、Prompt。
- `AdvancedTeamConfig`：高级折叠区。

`BotEdit` 可以重构成更小的数据表单单元，或由 `BotCoreConfigForm` 替换。保存逻辑应抽到可测试的 helper 或 hook 中，避免简洁界面和高级界面复制 Bot 保存逻辑。

## 数据流

新建智能体默认值：

- `workflow.mode = "solo"`
- `bind_mode = ["chat"]`
- 一个待创建 Bot
- 执行器默认优先选择 Chat Shell

保存简洁配置时：

1. 校验智能体名称。
2. 校验至少一个绑定模式。
3. 根据执行器选择解析目标 Shell。
4. 校验模型、Prompt、技能、知识库配置。
5. 创建或更新 Bot。
6. 创建或更新 Team，绑定单个 Bot，角色为 `leader`。
7. 刷新 `TeamContext`。

编辑已有复杂协作智能体时：

- 默认展开高级配置，并提示该智能体使用复杂协作模式。
- 不自动把复杂协作智能体降级为 solo，除非用户明确选择切换到 solo 并确认。

## 错误处理

- 没有可用 Chat Shell 时，`简单`置灰并说明需要先配置执行器。
- 没有可用 ClaudeCode Shell 时，`复杂`置灰并说明编码或设备任务需要 ClaudeCode。
- 绑定模式包含 `编码` 或 `设备` 时，如果当前执行器不兼容，自动切换到 `复杂`，无法切换时提示用户配置执行器。
- 自定义 Shell 未选择时，保存前提示选择具体 Shell。
- 保留现有 API 错误 toast，但前端先做清晰校验。

## 测试

先补测试再实现：

- 新建智能体默认 `bind_mode` 为 `["chat"]`，默认保存 `workflow.mode = "solo"`。
- 绑定模式只渲染 `对话`、`编码`、`设备`，并且选中态可被测试识别。
- 选择 `编码` 或 `设备` 时，执行器限制到 ClaudeCode 类配置。
- 简洁保存会创建或更新单个 Bot，并把 Team 的 Bot role 设置为 `leader`。
- 模型、技能、知识库和 Prompt 会进入 Bot 保存 payload。
- 非 solo 的已有智能体不会在打开简洁页时被静默降级。
- 高级入口仍可渲染协作模式和现有成员配置。

## 非目标

- 不修改后端 CRD 或数据库结构。
- 不新增新的 `workflow.mode`。
- 不删除现有协作模式。
- 不在本次改动中重做智能体列表页。
