# 技能库与使用位置产品设计

## 背景

当前技能交互把“上传技能”“选择智能体”“聊天时选择技能”混在一起。用户上传 Skill 后，不知道它到底属于自己、属于智能体，还是只是在技能库里；组 Skill 和系统 Skill 又会进一步放大这个混乱。

新的主心智改为：

> Skill 先进入技能库；再被加入某个使用位置；当前对话自动汇总这些使用位置里的 Skill。

这样可以同时解释：

- 上传 / 导入只是创建或获得一个 Skill 资产。
- “对我可用”是把某个 Skill 加到我的默认能力。
- “添加到智能体”是把某个 Skill 变成智能体自带能力。
- 未来“添加到项目”是把某个 Skill 变成项目能力，类似 Claude Code 的 project skill。
- 聊天里仍然可以本次临时使用一个 Skill。

## 核心目标

- 让用户上传或导入 Skill 后，能马上理解它已经进入哪里。
- 让新上传个人 Skill 符合“上传了就能用”的直觉。
- 清楚区分 Skill 的资产归属和使用位置。
- 组 Skill 不被错误理解成“我的 Skill”，但可以被某个用户加入自己的默认能力。
- 不要求用户为了使用 Skill 复制系统默认智能体。
- 保留聊天中的本次临时使用能力。
- 为后续项目级 Skill 留出清晰扩展点。

## 非目标

- 第一版不做完整项目级 Skill。
- 第一版不做复杂例外规则。
- 不把“对我可用”写入智能体本体。
- 不把“添加到智能体”理解为当前用户自己的默认能力。
- 不让组 Skill 因为某个成员操作而自动影响全组成员。

## 三层模型

```text
Skill 资产
  一个可复用能力包
  有归属：个人 / 组 / 系统 / 资源库

使用位置
  某个 Skill 被放到哪里长期使用
  包括：我的默认能力 / 智能体自带能力 / 项目能力 / 本次临时使用

运行时生效
  当前对话最终带上的 Skill 集合
  来自：智能体自带 + 项目能力 + 我的默认能力 + 本次临时使用
```

一句话：

> 技能库解决“我有哪些 Skill”；使用位置解决“这些 Skill 在哪里会被用”。

## 关键概念

### 技能库

技能库是用户能看到、管理、使用的 Skill 资产集合。

技能库里可以有：

- 我上传的 Skill。
- 我导入或安装的 Skill。
- 我所在组里的 Skill。
- 系统或资源库提供的 Skill。

技能库里的 Skill 不等于自动生效。它只是一个可被加入不同使用位置的资产。

### 我的默认能力

“我的默认能力”是一组当前用户自己的长期使用配置。

含义：

```text
这个 Skill 会跟着我走。
我使用默认助手、系统助手、共享助手、自己创建的助手时，它都自动可用。
```

边界：

- 只影响当前用户。
- 不修改系统助手。
- 不修改共享助手。
- 不修改智能体的 Ghost 配置。
- 可以引用个人 Skill，也可以引用我有权限使用的组 Skill 或系统 Skill。

### 智能体自带能力

“智能体自带能力”是智能体定义的一部分，本质上是 Ghost 绑定 Skill。

含义：

```text
这个 Skill 跟着智能体走。
智能体被分享、发布、复制时，它作为智能体能力的一部分存在。
```

边界：

- 需要用户有编辑智能体的权限。
- 不等于把这个 Skill 加入“我的默认能力”。
- 系统或他人共享的智能体，如果用户没有编辑权，就不能直接改它的自带能力。

### 项目能力

“项目能力”是后续扩展，用于表达某个 Skill 只在项目内自动可用。

含义：

```text
这个 Skill 跟着项目走。
只要当前对话或任务在这个项目里，它就自动可用。
```

第一版只保留概念和数据模型扩展方向，不实现完整项目能力。

### 本次临时使用

“本次临时使用”只影响当前这一次发送。

含义：

```text
这个 Skill 只参与这条消息。
发送后清空。
不加入我的默认能力。
不添加到智能体。
不添加到项目。
```

## 归属与使用位置

Skill 有两个不同问题，必须分开：

```text
归属
  这个 Skill 资产是谁的

使用位置
  这个 Skill 被放到哪里自动使用
```

示例：

| Skill 来源 | 资产归属 | 可以加入我的默认能力 | 可以添加到智能体 | 说明 |
| --- | --- | --- | --- | --- |
| 我上传的 Skill | 我 | 可以 | 可以，若我能编辑智能体 | 新上传后默认加入我的默认能力 |
| 组里的 Skill | 组 | 可以，若我有使用权限 | 可以，若我能编辑目标智能体 | 不属于我，但可以对我可用 |
| 系统 Skill | 系统 | 可以，若系统允许用户使用 | 通常由系统或管理员配置 | 不会默认变成我的能力 |
| 资源库 Skill | 资源库 / 原作者 | 安装或导入后可加入 | 可按权限添加 | 安装后进入技能库 |

组 Skill 的核心解释：

> 组 Skill 不是“我的 Skill”，而是“我可用的 Skill”。我可以把它加入我的默认能力，但这个动作只影响我，不影响其他组员，也不改变组 Skill 资产本身。

## 使用位置关系

产品上应该把“使用位置”理解为引用关系，而不是 Skill 资产自己的字段。

建议抽象：

```text
SkillUsage
  skillRef
  targetType: user | agent | project | message
  targetId
  ownerUserId
  createdBy
```

解释：

- `targetType=user`：加入我的默认能力。
- `targetType=agent`：添加到智能体自带能力。
- `targetType=project`：添加到项目能力。
- `targetType=message`：本次临时使用。

这样组 Skill 可以被多个用户分别加入自己的默认能力，而不会修改组 Skill 资产。

## 兼容规则

为避免影响现有系统：

```text
存量 Skill 资产：只保留在技能库，不自动加入我的默认能力
新上传个人 Skill：进入技能库，并默认加入上传者的我的默认能力
新导入个人 Skill：进入技能库，并默认加入导入者的我的默认能力
新上传组 Skill：进入组技能库，并默认只加入上传者的我的默认能力
系统 / 公共 Skill：不自动加入任何用户的默认能力
智能体已有 Skill：继续作为智能体自带能力生效
聊天已选 Skill：继续作为本次临时使用生效
```

这里的“默认加入”是产品动作，不代表 Skill 资产本身有全局 `enabled=true`。

## 上传后交互

上传成功后展示两个结果：

```text
Skill 已加入技能库
已加入“我的默认能力”，之后对话会自动可用。

[知道了] [管理使用位置]
```

如果上传到组：

```text
Skill 已加入组技能库
已为你加入“我的默认能力”，其他组员不会受影响。

[知道了] [管理使用位置]
```

说明文案：

```text
“我的默认能力”只影响你的对话，不会修改助手本身，也不会影响其他用户。
```

用户关闭反馈后，仍然能在技能库列表、详情页和聊天技能按钮里看到同样含义。

## 技能库页面

页面标题建议从“我的技能”改为“技能库”。

列表展示：

```text
技能名称
描述
归属：我 / 组 / 系统 / 资源库
使用位置：我的默认能力 / 智能体自带 / 项目能力
操作：加入我的默认能力 / 移出我的默认能力 / 添加到智能体 / 详情
```

列表顶部长期说明：

```text
技能库保存你可用的 Skill。加入“我的默认能力”后，它会在你的对话中自动可用；也可以添加到智能体或在聊天时本次临时使用。
```

每个 Skill 行要同时展示归属和使用位置：

```text
组 Skill · 设计团队
已加入我的默认能力 · 会在你的对话中自动可用
```

```text
系统 Skill
未加入我的默认能力 · 可在聊天时本次临时使用
```

空状态：

```text
还没有可用 Skill
上传、导入或从资源库安装一个 Skill 后，它会出现在这里。
```

## 聊天技能按钮

聊天输入框保留技能按钮，但定位改为：

```text
查看自动可用 Skill + 本次临时使用 Skill
```

打开后分两块：

```text
自动可用
  助手自带能力
  项目能力（未来）
  我的默认能力
  只读展示，不在这里做长期配置

本次临时使用
  技能库中当前没有自动进入对话的 Skill
  可以选择几个只用于这条消息
  发送后清空
```

底部入口：

```text
管理技能库
```

长期说明：

```text
自动可用的 Skill 不需要每次选择。
本次临时使用只影响这条消息。
```

当“自动可用”为空：

```text
暂无自动可用 Skill
你可以在技能库中加入“我的默认能力”，或在下方本次临时使用。
```

临时使用后，输入框上方持续显示：

```text
本次临时使用：Markdown 表格整理
只影响这条消息，发送后不会加入默认能力。
```

如果用户频繁临时使用同一个 Skill，可以提示：

```text
经常使用这个 Skill？
[加入我的默认能力]
```

## 聊天页展示

聊天页主选择仍然是智能体。Skill 只是能力说明，不和智能体成为同级主选择。

建议摘要：

```text
当前助手：默认助手
Skill：自动可用 3 个，本次临时使用 1 个
```

展开后按来源展示：

```text
助手自带能力
项目能力
我的默认能力
本次临时使用
```

去重规则：

- 同一个 Skill 资产只展示和下发一次。
- 展示优先级：助手自带能力 > 项目能力 > 我的默认能力 > 本次临时使用。
- 如果同名但不同来源的 Skill 冲突，必须用明确的 skillRef 区分，不能只靠名称。

## 智能体编辑页

智能体编辑页里的 Skill 区域不要叫“启用 Skill”，建议叫：

```text
智能体自带能力
```

说明：

```text
添加到这里的 Skill 会跟随这个智能体。它不是你的默认能力，也不会自动影响其他智能体。
```

从智能体编辑页添加 Skill 时，选择器从技能库中选 Skill 资产。

对于用户不能编辑的系统智能体或他人共享智能体：

- 不展示“添加到智能体”操作。
- 可以引导用户把相关 Skill 加入“我的默认能力”。
- 也可以允许本次临时使用。

## 产品规则

1. 上传 / 导入 / 安装只创建或获得 Skill 资产。
2. 新上传或新导入个人 Skill 默认加入上传者 / 导入者的我的默认能力。
3. 存量 Skill 不自动加入我的默认能力。
4. 组 Skill 的归属是组，不是个人。
5. 用户可以把有权限使用的组 Skill 加入自己的默认能力。
6. 加入我的默认能力只影响当前用户。
7. 添加到智能体会改变智能体自带能力，需要编辑权限。
8. 本次临时使用只影响当前消息。
9. 运行时按使用位置汇总 Skill，并按 skillRef 去重。

## 推荐文案

- 使用“技能库”，不用“我的技能”作为总入口。
- 使用“加入我的默认能力”，不用“启用到我”。
- 使用“移出我的默认能力”，不用“关闭技能”。
- 使用“智能体自带能力”，不用“智能体启用技能”。
- 使用“本次临时使用”，不用“临时启用”。
- 使用“归属”，解释 Skill 资产是谁的。
- 使用“使用位置”，解释 Skill 会在哪里自动可用。

## 长期界面引导

这套设计的引导是长期界面能力，不是一次性教程或 toast。用户每次看到 Skill 相关界面时，都应该能从界面结构和操作文案理解：

- 这个 Skill 资产属于谁。
- 它是否已经加入我的默认能力。
- 它是否是当前智能体自带能力。
- 它是否只是本次临时使用。
- 对我可用不等于修改智能体。
- 组 Skill 不等于我的 Skill。

长期引导由以下界面元素承担：

- 分区标题：`技能库`、`我的默认能力`、`智能体自带能力`、`本次临时使用`。
- 归属标签：`我上传`、`组 Skill`、`系统 Skill`、`资源库`。
- 使用位置标签：`已加入我的默认能力`、`智能体自带`、`本次`。
- 空态说明：告诉用户为什么为空以及下一步能做什么。
- 操作文案：`加入我的默认能力`、`添加到智能体`、`本次临时使用`。

## 第一版范围

第一版做：

- 技能库里区分 Skill 归属和使用位置。
- 增加“我的默认能力”使用位置。
- 新上传 / 新导入个人 Skill 默认加入我的默认能力。
- 存量 Skill 默认只在技能库中，不自动加入我的默认能力。
- 组 Skill 可以由当前用户加入自己的默认能力，但不影响其他组员。
- 聊天技能按钮拆成“自动可用”和“本次临时使用”。
- 自动可用中展示助手自带能力和我的默认能力。
- 通过状态、副标题、空态、归属标签和使用位置标签提供持续解释。

第一版不做完整项目能力；只保留数据和文案扩展空间。
