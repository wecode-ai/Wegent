# 技能生效范围产品设计

## 背景

当前技能交互把“上传技能”“选择智能体”“聊天时选择技能”混在一起。用户上传技能后，系统默认智能体不能直接使用，必须新建或编辑智能体才能获得类似体验；但默认智能体本身已有系统内置技能，用户很难复刻一个完整的“带技能智能体”。

问题本质不是缺少入口，而是概念关系不清晰：

- 智能体是聊天和任务的主要使用对象。
- 技能是可复用能力，不应该成为聊天时和智能体并列的主选择项。
- 用户需要一个长期配置方式，让技能在合适场景自动生效。

## 目标

- 让用户上传或安装技能后，能明确设置它在哪里长期生效。
- 聊天时不要求用户每次选择技能，默认根据当前上下文自动带上已生效技能。
- 保留临时追加能力，满足本次请求需要额外技能的场景。
- 避免用户被“我、技能、智能体”三个概念绕晕。
- 不要求用户复制系统默认智能体。

## 非目标

- 本轮不引入项目级技能作用域。
- 本轮不重做智能体创建流程。
- 本轮不取消系统或智能体内置技能。
- 本轮不把聊天里的临时追加变成长期配置的唯一入口。

## 核心定义

技能不是聊天时主要选择项，而是一个可长期配置的能力。

产品规则是一句话：

> 平时不用选技能；技能根据生效范围自动生效；本次特殊需要时可以临时追加。

概念关系如下：

```text
我
  拥有技能，管理技能的生效范围

技能
  是可复用能力
  可以长期生效，也可以本次临时追加

智能体
  是聊天和任务的主要使用对象
  决定当前上下文和场景

聊天输入框
  展示当前已生效技能
  支持本次临时追加
  引导用户去做长期配置
```

## 生效范围

技能配置页为每个技能提供“生效范围”设置。第一版支持三种范围：

第一版采用“单一策略，多选对象”：

- 一个技能只能选择一种生效策略：所有智能体、指定场景、指定智能体。
- 选择“指定场景”时，可以勾选多个场景。
- 选择“指定智能体”时，可以勾选多个智能体。
- “所有智能体”只对当前用户自己的使用体验生效，不修改系统智能体本体，也不影响其他用户。

### 所有智能体

适合通用技能，例如翻译、总结、格式转换。

文案：

```text
所有智能体
默认智能体和你创建的智能体都会使用
```

### 指定场景

适合只在某类任务里有用的技能。

第一版场景：

- 对话
- 编码
- 设备

文案：

```text
指定场景
只在选中的场景中自动生效
```

### 指定智能体

适合专门增强某几个智能体。

文案：

```text
指定智能体
只在使用选中的智能体时自动生效
```

## 技能状态

技能列表不再只显示“已上传”。建议显示技能的生效状态：

```text
未设置生效范围
所有智能体生效
指定场景生效：对话、编码
指定智能体生效：默认助手、代码助手
```

“未设置生效范围”表示技能已在我的技能库中，但不会自动进入任何请求。它仍然可以在聊天中临时追加。

## 上传后引导

上传成功不是流程结束，应立即引导用户设置生效范围。

弹窗内容：

```text
技能已上传。你想让它在哪里生效？

[所有智能体] [指定场景] [指定智能体] [稍后设置]
```

默认推荐：

- 从普通聊天入口上传时，推荐“所有智能体”。
- 从智能体配置页上传时，推荐“指定智能体”，并默认选中当前智能体。
- 从编码或设备相关入口上传时，推荐“指定场景”，并默认选中当前场景。

## 聊天技能按钮

聊天输入框仍保留技能按钮，但定位从“选择技能”改为“查看和临时追加”。

打开后分两块：

```text
已生效技能
  当前会自动带上的技能
  只读，不可取消勾选
  显示来源：所有智能体 / 编码场景 / 默认助手

临时追加
  本次消息额外使用的技能
  可以勾选
  发送后不改变长期配置
```

底部提供长期配置入口：

```text
管理技能生效范围
```

当用户临时追加某个技能后，可以提供轻量动作：

```text
设为长期生效
```

该动作打开生效范围配置，不直接静默保存，避免用户误把临时选择变成全局行为。

## 聊天页心智

聊天页的主选择仍然是智能体。技能只是当前请求的能力说明。

建议展示：

```text
当前智能体：默认助手
技能：已生效 4 个，临时追加 1 个
```

如果没有任何技能生效：

```text
技能：无已生效技能
```

如果用户点开技能按钮，应先看到“已生效技能”，再看到“临时追加”。

## 产品规则

1. 已生效技能自动参与当前请求，不需要用户每次勾选。
2. 聊天里不能取消某个长期生效技能，只能去技能配置页修改生效范围。
3. 临时追加只影响本次发送。
4. 临时追加过的技能应提供转长期配置的引导。
5. 技能生效范围是当前用户自己的配置，不影响其他用户。
6. 系统或智能体内置技能可以展示为“系统内置”，但普通用户不在聊天技能按钮里编辑它们。
7. 同一个技能如果被多个范围命中，只展示一次，并显示最清晰的来源摘要。

## 推荐文案

- 使用“生效范围”，不用“启用范围”。
- 使用“已生效技能”，不用“已选择技能”。
- 使用“临时追加”，不用“未生效技能”。
- 使用“设为长期生效”，不用“固定技能”。
- 使用“管理生效范围”，不用“管理技能绑定”。

## 体验示例

### 上传并长期生效

1. 用户上传“周报总结”技能。
2. 上传成功弹窗询问生效范围。
3. 用户选择“所有智能体”。
4. 后续使用默认助手或自定义智能体时，“周报总结”出现在已生效技能中。

### 本次临时追加

1. 用户在聊天中点开技能按钮。
2. “已生效技能”展示当前自动生效的技能。
3. 用户在“临时追加”中勾选“Markdown 表格整理”。
4. 本次发送时带上该技能。
5. 发送后临时追加清空。
6. 如果用户经常追加该技能，界面提示“设为长期生效”。

### 修改长期配置

1. 用户进入“我的技能”。
2. 找到“Markdown 表格整理”。
3. 打开“生效范围”。
4. 选择“指定场景：对话”。
5. 后续只有对话场景自动带上该技能。

## 第一版范围

第一版只做三件事：

- 技能配置页支持生效范围配置。
- 上传后增加生效范围引导。
- 聊天技能按钮拆成“已生效技能”和“临时追加”，并提供管理入口。

第一版暂不引入项目级作用域。等“所有智能体 / 指定场景 / 指定智能体”跑顺后，再评估项目技能。
